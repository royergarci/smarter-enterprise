Definition

An icon is a graphic symbol (usually a simple picture) that denotes a program, command, data file or concept.

Overview

Using common icons on IBM web pages improves site performance, content consistency and general usability.

Requirements

Pages that do not adhere to the following requirements may be delayed or disallowed.

  • Use only the approved icons detailed on this page. To request a new icon, post a question to the w3 and www Web Design Standards community.
  • All icons must be referenced from the appropriate cascading style sheet (CSS). Do not use embedded images.
  • Provide descriptive alternative tag information for all icons.
  • All icon labels and text links must use sentence style capitalization, unless the text contains a proper noun or other word that requires capitalization (e.g., acronyms).
  • Always use the standard icons (excluding anchor links) with a white background. Placing icons on other background colors requires special approval. To request approval, post a request to the w3 and www Web Design Standards community.
  • When linking to content outside of IBM, notify users (in parenthesis) that they are connecting to a location outside of IBM?s site domain.
  • Links to content outside of IBM web pages not embedded within a sentence or a paragraph must be preceded by the standard external link (class="ibm-external-link") icon. For more information about external links, see the External links standard.
  • Content that is entitled must be grouped or identified with a key icon.
    • Entitled content should be treated as a regular link, with the default link color defined by the css.
    • Users accessing entitled content must sign in with their user ID and password.
  • IBM only has permission to use social media icons in certain use cases. Only use third-party social icons/thumbnails for the following approved use cases:
    • Use case 1: Linking users to third-party sites for the purpose of engaging in social activity (e.g., viewing an IBMer/IBM group page on a third-party site)
    • Use case 2: Linking users to third-party sites for the purpose of social bookmarking via IBM social bookmarking service (SBS)
    • Use case 3: Inclusion in e-mail campaigns
    • Use case 4: Usage of the Flickr API in the IBM Social Aggregator. All other use of the Flickr API requires prior approval from both IBM and Flickr legal teams.
    • Note: You may not share pages or content from external social media sites like Facebook and LinkedIn on IBM web pages. For more information about sharing pages and content from social media sites, see Social sharing.
