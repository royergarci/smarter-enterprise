Template updates to enable v17e

  1. Add viewport meta tag below content type:
    <meta name="viewport" content="width=device-width, initial-scale=1"/>

  2. Replace current v17 ww.css file with new v17e version:

    Old:
    <link href="//1.www.s81c.com/common/v17/css/www.css" rel="stylesheet" title="www" type="text/css"/>

    New:
    <link href="//1.www.s81c.com/common/v17e/css/ww.css" rel="stylesheet" title="www" type="text/css" />

  3. Add new ibm-type class to <body> tag for latin based templates / languages (this class applies the custom fonts)

    e.g.
    <body id="ibm-com" class="ibm-type">
  4. If need, the data tables, forms, application, social icons for the content space and modules are now loaded separately.

    e.g.
    <link href="//1.www.s81c.com/common/v17e/css/data.css" rel="stylesheet" title="www" type="text/css" />

    <link href="//1.www.s81c.com/common/v17e/css/form.css" rel="stylesheet" title="www" type="text/css" />

    <link href="//1.www.s81c.com/common/v17e/css/social.css" rel="stylesheet" title="www" type="text/css" />

    <link href="//1.www.s81c.com/common/v17e/css/application.css" rel="stylesheet" title="www" type="text/css" />

    <link href="//1.www.s81c.com/common/v17e/css/module.css" rel="stylesheet" title="www" type="text/css" />

  5. The separate module.css file covers:

    -- anchor link A-Z (.ibm-link-list-az)
    -- directory link list (.ibm-directory-link-list)
    -- list container (.ibm-list-container)
    -- article list (.ibm-article-list)
    -- twitter list and modules (.ibm-twitter-list / .ibm-twitter-module)
    -- twisties (.ibm-twisty)
    -- RSS module (.ibm-rss-module)
    -- step indication and container (.ibm-step-indicator / .ibm-step-container)

  6. jQuery and Modernizr are included as part of the common v17e JS

    1.www.s81c.com/common/v17e/js/ww.js
