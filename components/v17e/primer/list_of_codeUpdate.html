Requirements to Implement v17e

Follow the steps below to change your page to v17e.

For all pages

  • Add viewport meta tag below content type meta tag: 
    		<meta name="viewport" content="width=device-width, initial-scale=1">
  • Replace current v17 www.css file with new v17e version 
    <link href="//1.www.s81c.com/common/v17e/css/ww.css" rel="stylesheet" title="www" type="text/css" />
  • Replace current v17 www.js file with v17e version. jQuery and Modernizr are included as part of the common v17e JS 
    <script src="//1.www.s81c.com/common/v17e/js/ww.js" type="text/javascript"></script>

Optional Requirements

To make the v17e CSS load faster, some of the CSS code has been removed from the main CSS file. As such some additional CSS file(s) may need to be called based on page content.

  • Application/Slim Masthead 
    <link href="//1.www.s81c.com/common/v17e/css/application.css" rel="stylesheet" title="www" type="text/css"/>
  • Tables 
    <link href="//1.www.s81c.com/common/v17e/css/data.css" rel="stylesheet" title="www" type="text/css" />
  • Forms 
    <link href="//1.www.s81c.com/common/v17e/css/forms.css" rel="stylesheet" title="www" type="text/css" />
  • Social Icons for the content area 
    <link href="//1.www.s81c.com/common/v17e/css/social.css" rel="stylesheet" title="www" type="text/css" />
  • Modules : The following CSS file needs to be called if any of the following classes are used
    • anchor link A-Z (.ibm-link-list-az)
    • directory link list (.ibm-directory-link-list)
    • list container (.ibm-list-container)
    • article list (.ibm-article-list)
    • twitter list and modules (.ibm-twitter-list / .ibm-twitter-module)
    • twisties (.ibm-twisty)
    • RSS module (.ibm-rss-module)
    • step indication and container (.ibm-step-indicator / .ibm-step-container)
    <link href="//1.www.s81c.com/common/v17e/css/module.css" rel="stylesheet" title="www" type="text/css" />

This class is a mandatory class required for 5-column layout(page with left navigation). There is no change in the UI with this class. This class is introduced to make the 5 column layout responsive.

.ibm-thumbnail-module, .ibm-thumbnail-smaill, .ibm-thumbnail-swap, .ibm-sprite

  • .ibm-thumbnail-module - Applied within columns to allow hover / click based on sprites. This can help draw a visitors attention due to the animation and responsive nature of the experience. This container also changes the layout under the 480 grid
  • .ibm-thumbnail-small - Applying "ibm-thumbnail-small" class to the "ibm-columns" container will automatically convert a 4 col large image into 2 x 2 rows of smaller thumbnails
  • .ibm-thumbnail-swap - Applied to ibm-thumbnail-module to swap image to right side. This applies to mobile (480) grid.
  • .ibm-sprite - Applied to container to add image sprites for mouse hover states

Text treatment classes

V17e introduced few classes to achieve different size and alignment for text.

  • .large: Applied to column grid (4-2,4-3,5-2,5-3,5-4,6-3,6-4,6-5)
  • .small: Applied to column grid (4-2,4-3,5-2,5-3,5-4,6-3,6-4,6-5)
  • .center: Applied to align the content , headers to the center
  • .bottom: Applied to align the content at the bottom, when full width image is used
  • .section-header: Applied to section headers
