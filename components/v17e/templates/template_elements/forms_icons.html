Overview

A web form on a web page allows users to enter data that is sent to a server for processing. Forms are a critical type of communication between users and IBM (for example, for providing contact information and feedback, for purchasing products or for managing inventory). They should always be designed with users' interests in mind, so that they are as easy to use and to understand as possible.

Note: When implementing forms, the classes "ibm-styled-form" and "ibm-styled" are optional CSS elements used primarily for implementing Dojo-styled form elements. Use these classes with caution, as they have some design limitations. For example:

  • You cannot use a select box inside a data table using either "styled" class.
  • Neither class can be used for advanced forms.
  • These classes convert any classic <input /> and <select /> fields to Dojo input controls. Therefore, they overwrite or remove any custom javascript event handlers you might have for an input field (for example, onclick, onchange, etc.) and most other custom javascript logic which acts on those fields will not work. If you would still like to use these classes, convert any javascript you use for these fields to Dojo code instead, to get a handle to the Dojo widgets (using dijit functions).
